线程池与CPU核心数的关系

经验上：N为CPU核心数

  • 如果是CPU密集型应用，线程池大小应该设置为N+1
  • 如果是IO密集型应用，线程池大小设置为2N+1，eg：数据库数据交互、文件上传下载

+1的目的：即使当线程偶尔由于缺失故障或者其他原因而暂停的时候，这个额外的线程也能够确保CPU的时钟周期不会被浪费。
